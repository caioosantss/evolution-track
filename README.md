# evolution-track
Este repositório é um **diário vivo de aprendizado**. Cada pasta é um módulo temático, cada commit é um passo, e cada branch é uma chance de olhar para trás e reescrever melhor o que eu já fiz.

A ideia não é ter código perfeito desde o início — é ter código que **evolui junto comigo**.
Além disso, tratarei este repositório como uma forma de criar "intimidade" com a plataforma Github. Usarei terminal, chamarei outros para colaborar, etc.

##  Evolução dos exercícios

Alguns exercícios serão revisitados conforme novos conhecimentos forem adquiridos.

A ideia é permitir que um mesmo problema evolua ao longo do tempo.  
Por exemplo: um sistema desenvolvido inicialmente com POO pode ser posteriormente adaptado para utilizar banco de dados ou ser exposto como uma API.

Essa evolução pode ocorrer por meio de:
- refatorações no próprio código
- novas versões organizadas por estrutura
- uso de branches para experimentação

---

##  Organização das Issues

O repositório utiliza issues como ferramenta de planejamento e acompanhamento do aprendizado, divididas em dois tipos:

- **Issues centrais**: relacionadas ao roadmap geral (ex: estudar um conceito ou tecnologia)
- **Issues específicas**: relacionadas à implementação dentro de exercícios

Issues específicas são agrupadas em **milestones**, facilitando a identificação do exercício ao qual pertencem e permitindo acompanhar o progresso de forma clara.

---

##  Padrão de Labels

As issues utilizam labels para facilitar organização e filtragem:

### Áreas
- `poo` → Programação Orientada a Objetos  
- `sql` → Banco de dados  
- `api` → Desenvolvimento de APIs  
 

### Tipo
- `study` → Estudo teórico  
- `task` → Implementação prática  
- `refactor` → Melhoria de código  
- `docs` → Documentação  
   

Essa estrutura permite visualizar claramente o progresso, separar teoria de prática e navegar pelo repositório de forma eficiente.
